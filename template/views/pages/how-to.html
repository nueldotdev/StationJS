How StationJS Works

1. Defining Routes with route

The route function is the core of StationJS. It allows you to define individual routes and their handlers. Here's an example:

import { route } from 'station-js';

const homeRoute = route('/home', {
  GET: (ctx) => {
    return { body: { message: 'Welcome to StationJS!' }, status: 200 };
  }
});

Each route can handle multiple HTTP methods (e.g., GET, POST, DELETE) by defining them as keys in the handler object.

2. Grouping Routes with use

StationJS allows you to group related routes under a base path using the use function:

import { use, route } from 'station-js';

const userRoutes = use('/users', [
  route('/', {
    GET: (ctx) => {
      return { body: { message: 'List of users' }, status: 200 };
    },
    POST: (ctx) => {
      return { body: { message: 'User created' }, status: 201 };
    }
  }),
  route('/@id', {
    GET: (ctx) => {
      return { body: { message: `User with ID: ${ctx.params.id}` }, status: 200 };
    }
  })
]);

The @id syntax allows you to define dynamic parameters in your routes.

3. Serving HTML Pages

StationJS supports file-based routing for serving HTML pages. By default, it looks for files in the views directory. For example:

// Accessing http://localhost:3000/about
// Serves the file views/about.html

You can customize the directory by passing options when creating the server.

4. Hot Reloading

StationJS includes a hot reloading feature to automatically restart the server when you make changes to your code:

station start

This ensures a smooth development experience without manual restarts.

5. Customizing Static and Media Directories

You can customize where StationJS looks for static files (e.g., CSS, JS) and media files (e.g., images) by providing options:

import { createServer } from 'station-js';

const options = {
  publicDir: 'views', // Directory for HTML files
  staticDir: 'static', // Directory for CSS/JS files
  mediaDir: 'media' // Directory for images/videos
};

const server = createServer(routes, options);
server.listen(3000);

With this setup, you can organize your project however you like.

6. Putting It All Together

Here's a complete example of a StationJS server:

import { createServer, route, use } from 'station-js';

const homeRoute = route('/home', {
  GET: (ctx) => {
    return { body: { message: 'Welcome to StationJS!' }, status: 200 };
  }
});

const userRoutes = use('/users', [
  route('/', {
    GET: (ctx) => {
      return { body: { message: 'List of users' }, status: 200 };
    }
  })
]);

const routes = [homeRoute, ...userRoutes];

const options = {
  publicDir: 'views',
  staticDir: 'static',
  mediaDir: 'media'
};

const server = createServer(routes, options);
server.listen(3000, () => {
  console.log('Server is running on http://localhost:3000');
});
